**3.1.2 로그인**

- 로그인은 **SNS 로그인 화면**과 **이메일 로그인 화면**으로 나눠져 있습니다.
  - SNS(GitHub) 로그인은 구현하지 않으며, 화면에 버튼만 배치하도록 합니다.
- 이메일과 비밀번호를 모두 입력하면 `로그인` 버튼이 활성화됩니다. 입력되지 않은 입력창이 있다면 버튼은 활성되지 않습니다.
- `로그인` 버튼을 클릭하면 이메일 주소와 비밀번호에 대한 유효성 검사를 진행하며, 이메일 주소 또는 비밀번호가 일치하지 않을 경우에는 경고 문구가 나타납니다.
- 로그인은 JWT(JSON Web Token) 기반 인증 방식을 사용합니다.
  - access_token: 1시간 유효
  - refresh_token: 7시간 유효

**3.1.3 회원가입**

- 로그인 화면에서 `이메일로 회원가입`을 누르면 회원가입 화면이 나타납니다.
- 회원가입은 **이메일 인증** → **비밀번호 설정 → 로그인 → 프로필 설정** 순서로 진행됩니다.
- **이메일 인증**
  - 이메일 주소를 입력하고 `인증번호 발송` 버튼을 클릭하면 이메일로 인증번호가 발송됩니다.
  - **실제 이메일이 발송되지 않습니다.** 올바른 이메일 인증 코드는 `123456`으로 고정되어 있습니다.
  - 이메일 형식이 유효하지 않거나 이미 가입된 이메일일 경우 경고 문구가 나타납니다.
  - 인증번호를 올바르게 입력하면 `다음` 버튼이 활성화됩니다.
- **비밀번호 설정**
  - 비밀번호와 비밀번호 확인을 모두 입력합니다.
  - 비밀번호가 6자 미만일 경우 경고 문구가 나타납니다.
  - 비밀번호가 일치하지 않는 경우 경고 문구가 나타납니다.
  - 작성이 완료된 후, 유효성 검사를 통과할 경우 `동의하고 회원가입` 버튼이 활성화 됩니다.
  - 버튼을 클릭하면 회원가입 완료 화면이 나타나며, 로그인하기 버튼을 클릭하면 로그인 화면으로 이동합니다.
  - 회원가입 완료 시 **준회원**으로 가입되며, 프로필이 완료되면 정회원으로 자동으로 변경됩니다.
- **프로필 설정**
  - 최초 로그인 시 프로필 생성화면이 나타납니다.
  - 프로필 사진, 닉네임, 소개, 이름, 휴대폰 번호, 선호 지역, GitHub 사용자명, 관심 태그를 입력받습니다.
    - 이름, 전화번호는 필수값입니다.
    - **닉네임과 전화번호는 고유해야 합니다.** (중복 불가)
  - 프로필 사진은 등록하지 않을 경우 기본 이미지가 등록되게 합니다.
  - 닉네임은 중복이 불가합니다. 닉네임 중복 확인 API를 사용하여 검사합니다.
  - 선호 지역은 미리 정해진 지역 목록 중에서 선택합니다.
  - 관심 태그는 여러 개 선택할 수 있습니다. (예: Python, JavaScript, React 등)
  - 프로필 설정을 완료하면 **정회원**이 되어 스터디 생성 및 참가가 가능해집니다.
  - 프로필 설정을 하지 않으면 **준회원** 상태로, 스터디 생성 및 참가가 제한됩니다.

**3.1.4 스터디인 메인 피드(홈 화면)**

- 스터디인 메인 피드는 메인 배너, 프로필, 참여 중인 스터디, 스터디 둘러보기 등이 표시되는 페이지입니다.
- 스터디 목록은 카드 형태로 표시되며, 스터디 카드에는 다음 정보가 포함됩니다.
  - 스터디 썸네일
  - 스터디 제목
  - 온/오프라인 여부
  - 스터디 장소 (오프라인인 경우)
  - 난이도 (초급/중급/고급)
  - 주제
  - 스터디 상태 (모집 중/모집 완료/진행 중/종료)
  - 참가자 수
  - 좋아요 여부
- 스터디가 없을 경우 "등록된 스터디가 없습니다" 문구가 표시됩니다.
- 스터디 목록은 페이지네이션을 지원합니다.
  - `page` 쿼리 파라미터로 페이지 번호 지정
  - `limit`, `skip` 쿼리 파라미터로 원하는 만큼 가져오기
  - 응답에 `count`, `next`, `previous`, `results` 필드 포함

**3.1.5 검색 및 필터**

<aside>
❗

**필수 과제에서는 검색 및 필터 기능은 구현하지 않습니다. 마크업만 구현해 보세요.**

</aside>

- 메인 피드 상단에 검색 창을 클릭하면 스터디 검색이 가능합니다. 검색어를 입력한 후 엔터를 누르면 검색 결과 페이지로 이동합니다.
- 스터디 제목, 소개, 태그로 검색할 수 있습니다.
- 필터 옵션
  - 온/오프라인(내 지역) 필터 (0: 온라인, 1: 오프라인(내 지역))
  - 요일 필터 (1~7: 월~일)
  - 주제 필터 (개념학습, 응용/활용, 프로젝트, 챌린지, 자격증/시험, 취업/코테, 특강, 기타)
  - 난이도 필터 (초급, 중급, 고급)
  - 스터디 상태 필터 (모집 중, 모집 완료, 진행 중, 종료)
- 검색 결과는 페이지네이션으로 표시됩니다.
- 검색 결과가 없는 경우에는 결과가 없음을 알리는 메세지와 스터디 만들기 버튼이 표시됩니다.

**3.1.6 스터디 상세 페이지**

- 스터디 목록에서 스터디를 클릭하면 상세 페이지가 나타납니다.
- 스터디 상세 페이지에서는 다음 정보를 확인할 수 있습니다:
  - 스터디 썸네일
  - 스터디 제목
  - 스터디장(그룹장) 정보 (닉네임, 프로필 이미지, 등급)
  - 온/오프라인 여부 및 장소 (오프라인일 경우)
  - 모집 인원
  - 스터디 소개
  - 진행 요일
  - 시작일
  - 진행 시간 및 기간
  - 모집 인원
  - 난이도
  - 주제
  - 검색 태그
  - 좋아요 수
  - 댓글 입력란과 댓글 리스트
- 스터디장의 프로필을 클릭하면 해당 사용자의 프로필 페이지로 이동합니다.

**3.1.7 스터디 생성**

- 메인 피드에서 `스터디 만들기` 버튼을 클릭하면 스터디 생성 페이지가 나타납니다.
- **정회원만** 스터디를 생성할 수 있습니다. 준회원일 경우 프로필 설정을 먼저 완료해야 합니다.
- 입력 항목
  - 스터디 썸네일 이미지 **(필수)**
  - 스터디 제목
  - 스터디 유형(온/오프라인) 선택 **(오프라인일 경우 지역 선택 필수)**
  - 모집 인원 (최소 3명, 최대 99명)
  - 스터디 소개, 스터디 일정, 그룹장 소개
  - 진행 요일 (월~일 중 복수 선택 가능)
  - 시작일
  - 기간 (주 단위)
  - 진행 시간 (시작 시간 ~ 종료 시간)
  - 주제 (개념학습, 응용/활용, 프로젝트, 챌린지, 자격증/시험, 취업/코테, 특강, 기타)
  - 난이도 (초급, 중급, 고급)
  - 검색 태그 (예: Python, Django)
- **모든 필수 입력(제목, 유형, 모집 인원, 시작일, 기간, 시간, 주제, 난이도, 검색 태그)**이 완료되면 `스터디 만들기` 버튼이 활성화됩니다.
- 스터디를 생성하면 자동으로 스터디장이 되며, 참가자 목록에 포함됩니다.

**3.1.7-1 AI 활용 기능 (스터디 콘텐츠 자동 생성)**

<aside>
❗

해당 기능은 AI 활용의 예시로 디자인은 제공하지 않습니다.

</aside>

스터디 생성 및 관리 시 AI를 활용하여 콘텐츠를 자동으로 생성할 수 있습니다.

**커리큘럼 자동 생성**

- 스터디 생성/수정 페이지에서 `AI 커리큘럼 생성` 버튼을 클릭하면 AI가 커리큘럼을 자동으로 생성합니다.
- 스터디 제목, 주제, 난이도, 기간, 진행 요일 등 입력된 스터디 정보를 기반으로 커리큘럼이 생성됩니다.
  - 필요한 정보가 입력되지 않은 경우 안내 문구를 표시합니다.
- 생성된 커리큘럼은 주차별(Week) 형태로 구성되며, 각 주차에 학습 주제와 목표가 포함됩니다.
- 사용자는 생성된 커리큘럼을 수정하거나 `다시 생성` 버튼으로 재생성할 수 있습니다.
- 최종 커리큘럼은 스터디 소개에 반영됩니다.

**스터디 소개글 자동 생성**

- 스터디 생성/수정 페이지에서 `AI 소개글 생성` 버튼을 클릭하면 AI가 스터디 소개글을 자동으로 생성합니다.
- 스터디 제목, 주제, 난이도, 유형(온/오프라인) 등 입력된 정보를 기반으로 소개글이 생성됩니다.
- 생성된 소개글은 스터디 소개 입력란에 자동으로 채워지며, 사용자가 직접 수정할 수 있습니다.
- `다시 생성` 버튼으로 새로운 소개글을 재생성할 수 있습니다.

**구현 요구사항**

- AI API 연동은 위니브에서 제공하는 Chat GPT API 명세([Chat GPT API 명세](https://www.notion.so/Chat-GPT-API-30cebf76ee8a806b996ac7e27c8f530e?pvs=21))를 사용합니다.
- AI 응답이 생성되는 동안 로딩 상태를 표시합니다 (스피너 또는 스켈레톤 UI).
- 스트리밍 응답을 활용하여 생성 중인 텍스트가 실시간으로 표시되도록 구현합니다. (선택)
- AI 생성 버튼은 해당 입력 필드 근처에 배치하며, 기존 입력 내용이 있을 경우 덮어쓰기 전에 확인을 요청합니다.

**3.1.8 스터디 수정/삭제**

- **스터디장만 스터디를 삭제할 수 있습니다.**
- 나의 스터디(내가 스터디장인 스터디) 상세 페이지에서 `수정` 버튼을 클릭하면 스터디 수정 페이지가 나타납니다.
- 스터디 생성과 동일한 입력 항목을 수정할 수 있습니다.
- 단, 모집 인원은 현재 참가 인원보다 적게 수정할 수 없습니다.
- 삭제 버튼을 클릭하면 확인 모달이 나타나고, 확인을 누르면 스터디가 삭제됩니다.

**3.1.9 사용자 프로필 페이지**

- 사용자 프로필 페이지에서는 다음 정보를 확인할 수 있습니다:
  - 프로필 이미지
  - 닉네임
  - 소개
  - 선호 지역
  - GitHub Contributions
  - 관심 태그
- 내 프로필을 조회할 경우 이름, 휴대폰 번호도 표시됩니다.
- 다른 사용자의 프로필을 조회할 경우 이름, 휴대폰 번호는 표시되지 않습니다.

**3.1.10 내 프로필 수정**

- 나의 프로필 페이지에서 `수정하기` 버튼을 클릭하면 프로필을 수정할 수 있는 페이지가 나타납니다.
- 입력창에 대한 명세는 회원가입에서의 프로필 설정과 동일합니다.
- 닉네임을 변경할 경우 중복 확인이 필요합니다.
- 변경 사항이 있고 유효성 검사가 통과될 경우 `저장하기` 버튼이 활성화됩니다.

**3.1.11 스터디 참가/탈퇴**

- 스터디 상세 페이지에 `참가하기` 버튼이 있습니다.
- **정회원만** 스터디에 참가할 수 있습니다.
- 이미 참가한 스터디일 경우 `탈퇴하기` 버튼이 표시됩니다.
- 스터디장은 스터디를 탈퇴할 수 없습니다.
- 모집 인원이 초과된 스터디에는 참가할 수 없습니다.
- 스터디 참가/탈퇴 시 채팅방에 시스템 메시지가 자동으로 생성됩니다.

**3.1.12 스터디 댓글 페이지**

<aside>
❗

**필수 과제에서는 대댓글 기능, 비밀 댓글 기능은 구현하지 않습니다.**

</aside>

- 스터디 상세 페이지 하단에 댓글 섹션이 있습니다.
- 댓글 입력창에 텍스트를 입력하면 `등록` 버튼이 활성화됩니다.
- **정회원만** 댓글을 작성할 수 있습니다.

**3.1.13 좋아요 버튼**

<aside>
❗

**필수 과제에서는 좋아요 기능은 구현하지 않습니다. 버튼의 변화만 구현하세요.**

</aside>

- 스터디가 나타나는 모든 페이지에 해당합니다.
- 스터디 카드 또는 상세 페이지에 하트 모양의 좋아요 버튼이 있습니다.
- 빈 하트를 클릭하면 색이 칠해진 하트로 변하고, 색이 칠해진 하트를 누르면 빈 하트로 변합니다.

**3.1.14 모달 버튼**

- 헤더 또는 스터디 상세 페이지 우측 상단에 더보기(⋮) 버튼이 있습니다.
- 버튼을 클릭하면 화면 하단에 모달이 나타납니다.
- **헤더**에 있는 버튼을 클릭하면 설정 및 개인정보와 로그아웃이 나타납니다.
- **스터디 상세 페이지**에서
  - 내가 만든 스터디일 경우 : 삭제, 수정 버튼이 나타납니다.
  - 다른 사용자가 만든 스터디일 경우 : 신고하기 버튼이 나타납니다.
- **댓글/대댓글**에서
  - 내가 작성한 댓글일 경우 : 삭제 버튼이 나타납니다.
  - 다른 사용자가 작성한 댓글일 경우 : 신고하기 버튼이 나타납니다.
- **로그아웃, 삭제, 신고 버튼**을 누르면 확인 메시지 모달창이 나타나야 하고, 취소 버튼을 누르면 모달은 사라집니다.

**3.1.14 배너**

- 메인 피드 상단에 배너 섹션이 있습니다.
- 배너는 이미지, 제목, 부제목, 배경색, 글자색, 링크로 구성됩니다.
- 배너를 클릭하면 해당 링크로 이동합니다.
