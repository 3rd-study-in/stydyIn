# DropdownSelect Atom 컴포넌트 제작 (2026-02-25)

## 만든 것

`src/atoms/DropdownSelect/` 폴더 아래 Figma 디자인 스펙을 기반으로 드롭다운/셀렉트 관련 UI 컴포넌트를 제작했다.

```
src/atoms/DropdownSelect/
├── _shared.jsx        공통 SVG 아이콘 + OptionList + useDropdown 훅
├── Dropdown.jsx       일반 드롭다운 (텍스트 + 화살표)
├── FilterDropdown.jsx 필터 칩 드롭다운 (필터 아이콘 + 레이블 + 화살표)
├── Select.jsx         아이콘 셀렉트 (텍스트 + 고정 우측 아이콘 슬롯)
└── index.js           3개 컴포넌트 일괄 re-export
```

---

## 컴포넌트별 역할

### Dropdown

`Dropdown.png` / `Dropdown-ON.png` 기반. 폼에서 값을 선택할 때 쓰는 일반적인 셀렉트 박스다. 닫힌 상태에서는 ▼, 열린 상태에서는 ▲ 화살표가 표시된다.

### FilterDropdown

`filter-dropdown-OFF.png` / `filter-dropdown-ON.png` 기반. 검색/필터 UI에서 쓰는 칩(chip) 형태의 드롭다운이다. 좌측에 깔때기 아이콘이 고정으로 붙고, 그 옆에 레이블 텍스트가 표시된다.

### Select

`Select-box-icon.png` 기반. Dropdown과 구조는 같지만 우측 아이콘이 상태(열림/닫힘)와 무관하게 고정된다. 달력 아이콘 등 커스텀 SVG를 `icon` prop으로 꽂는 슬롯 용도로 설계했다. SVG 파일이 아직 없어서 나중에 교체 예정이다.

## 설계 방식 선택 이유

### 1개 파일이 아닌 3개로 분리한 이유

처음에는 `variant` prop 하나로 분기하는 단일 파일(`DropdownSelect.jsx`)로 만들었다. 그런데 이 방식은 파일 하나가 여러 UI 형태를 동시에 책임지기 때문에, 한 variant를 수정할 때 다른 variant가 영향을 받는지 확인해야 하는 부담이 있다.

Figma 디자인에서도 이미지가 3종 세트로 나뉘어 있었기 때문에, 디자인 단위와 코드 단위를 맞추는 것이 다른팀원들이 확인할때 직관적이라고 판단해 3개로 분리했다.

### `_shared.jsx`를 둔 이유

3개 컴포넌트가 공통으로 사용하는 코드(SVG 아이콘, 옵션 리스트, open/close 훅)를 각 파일에 복사하면 수정 시 3곳을 모두 바꿔야 한다. `_shared.jsx`로 추출하면 한 곳만 수정하면 된다. `_` 접두사는 이 파일이 폴더 내부 전용이며 외부로 export되지 않는다는 관례적 표시다.

### SVG를 인라인으로 임시 구현한 이유

SVG 파일이 아직 없는 상태에서 Figma CSS의 `%` 좌표값을 `px`로 환산해 `<polygon>` / `<path>`로 직접 구현했다. 나중에 SVG 파일이 생기면 해당 컴포넌트의 `icon` prop에 꽂거나 `_shared.jsx`의 아이콘 함수만 교체하면 되도록 구조를 잡았다.

### 색상·글꼴을 arbitrary value로 쓴 이유

`src/index.css`의 `@theme` 블록은 아직 확정되지 않은 상태라 건드리지 않았다. 지금은 Tailwind의 arbitrary value(`text-[#8D9299]`, `border-[#D9DBE0]` 등)로 Figma 원본값을 직접 박아두었다. 나중에 디자인 시스템 토큰이 확정되면 `@theme`에 변수로 올리고 클래스만 교체하면 된다.

## 이로 인한 이점

- **파일 단위 책임 분리** — 디자인 3종과 파일 3개가 1:1로 대응되어 어떤 파일을 수정해야 하는지 즉시 알 수 있다.
- **공통 로직 단일 관리** — 옵션 리스트 hover 스타일, 외부 클릭 닫기, Escape 처리 등을 `_shared.jsx` 한 곳에서 관리한다.
- **SVG 교체 용이성** — 아이콘 함수가 명시적으로 분리되어 있어 SVG 파일로 교체할 때 수정 범위가 명확하다.
- **색상 변경 추적 용이** — arbitrary value로 박혀 있어서 에디터에서 hex값 검색만 해도 어디를 고쳐야 하는지 바로 찾을 수 있다.
- **배럴 export** — `index.js`를 통해 외부에서는 폴더 경로 하나로 3개 컴포넌트를 모두 import할 수 있다.

## 사용 예시

```jsx
import { Dropdown, FilterDropdown, Select } from '../atoms/DropdownSelect'

// 일반 드롭다운
<Dropdown
  options={[{ value: 'online', label: '온라인' }, { value: 'offline', label: '오프라인' }]}
  value={selected}
  onChange={setSelected}
  placeholder="스터디 유형 선택"
/>

// 필터 칩
<FilterDropdown
  options={[{ value: 'beginner', label: '초급' }, { value: 'mid', label: '중급' }]}
  value={filter}
  onChange={setFilter}
  label="난이도"
/>

// 달력 아이콘 셀렉트 (SVG 파일 생기면 icon에 꽂으면 됨)
<Select
  options={[...]}
  value={date}
  onChange={setDate}
  icon={<CalendarSVG />}
  placeholder="시작일 선택"
/>
```
