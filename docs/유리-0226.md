# 오늘 작업 (2026-02-26)

## 버튼 컴포넌트 제작

### 만든 것

`src/atoms/Button/` 폴더 아래 스터디인 서비스에서 사용될 버튼 컴포넌트들을 제작했다.

```
src/atoms/Button/
├── AlertButton.jsx    경고/확인 모달 등에 사용되는 버튼
├── Button.jsx         기본 버튼 컴포넌트
├── SideNavButton.jsx  사이드 네비게이션에 사용되는 버튼
└── index.js           3개 컴포넌트 일괄 re-export
```

### 컴포넌트별 역할

#### Button.jsx

서비스 전반에 걸쳐 사용될 가장 기본적인 형태의 버튼 컴포넌트. `props`를 통해 다양한 스타일과 동작을 유연하게 제어할 수 있도록 설계되었다.

#### SideNavButton.jsx

사이드 네비게이션 바에 특화된 버튼 컴포넌트. 네비게이션 항목으로서의 시각적 일관성과 클릭 동작을 담당한다.

#### AlertButton.jsx

주로 경고, 확인, 취소 등의 액션이 필요한 모달이나 팝업 내에서 사용되는 버튼. 특정 상황에서의 사용자 인터랙션을 명확하게 유도한다.

### 설계 방식 선택 이유

- **재사용성 및 일관성**: 서비스 내 다양한 버튼 요구사항을 충족시키면서도 일관된 UI/UX를 제공하기 위해 기본 `Button` 컴포넌트를 중심으로 확장하는 방식으로 설계했다.
- **역할 분리**: 각 버튼의 사용 목적과 위치에 따라 컴포넌트를 분리하여 코드의 가독성과 유지보수성을 높였다. 예를 들어, `SideNavButton`은 네비게이션에, `AlertButton`은 특정 액션에 집중하도록 했다.

### 이로 인한 이점

- **개발 효율성**: 공통된 스타일과 로직을 재사용하여 새로운 버튼이 필요할 때 빠르게 개발할 수 있다.
- **유지보수 용이성**: 특정 버튼의 스타일이나 동작 변경 시 해당 컴포넌트만 수정하면 되므로 유지보수가 용이하다.
- **디자인 일관성**: 모든 버튼이 정의된 컴포넌트를 통해 생성되므로 서비스 전체의 디자인 일관성을 유지할 수 있다.

## 헤더 컴포넌트 (검색 바 및 프로필 원) 제작

### 만든 것

헤더 영역에 들어갈 검색 바와 사용자 프로필 원 컴포넌트를 제작했다. 로그인 여부에 따른 프로필 로직도 함께 구현되었다.

```
src/atoms/SearchBar/
└── SearchBar.jsx      검색 입력 필드 및 검색 버튼을 포함하는 컴포넌트

src/atoms/ProfileCircle/
├── ProfileCircle.jsx  사용자 프로필 이미지를 표시하는 원형 컴포넌트
└── ProfileTest.jsx    ProfileCircle 컴포넌트의 로직 테스트를 위한 파일
```

### 컴포넌트별 역할

#### SearchBar.jsx

사용자가 스터디나 기타 정보를 검색할 수 있는 입력 필드와 검색 버튼을 제공한다. 헤더에 통합되어 전역 검색 기능을 담당한다.

#### ProfileCircle.jsx

사용자의 프로필 이미지를 원형으로 표시하는 컴포넌트. 로그인 상태에 따라 다른 이미지를 표시하거나, 클릭 시 사용자 메뉴 등의 추가 동작을 트리거할 수 있도록 설계되었다.

#### ProfileTest.jsx

`ProfileCircle.jsx` 컴포넌트의 다양한 상태(로그인/로그아웃, 이미지 유무 등)를 테스트하고 검증하기 위해 임시로 작성된 파일. 개발 과정에서 컴포넌트의 안정성을 확보하는 데 사용되었다.

### 설계 방식 선택 이유

- **모듈화**: 검색 바와 프로필 원을 각각 독립적인 컴포넌트로 분리하여 헤더뿐만 아니라 다른 영역에서도 재사용할 수 있도록 했다.
- **상태 기반 로직**: `ProfileCircle`의 경우, 사용자 로그인 상태에 따라 동적으로 UI를 변경해야 하므로, 관련 로직을 컴포넌트 내부에 캡슐화하여 관리의 용이성을 높였다.
- **테스트 용이성**: `ProfileTest.jsx`와 같은 테스트 파일을 통해 복잡한 상태 변화 로직을 개별적으로 검증할 수 있도록 했다.

### 이로 인한 이점

- **재사용성**: 검색 바와 프로필 원 컴포넌트를 헤더 외 다른 페이지나 섹션에서도 쉽게 활용할 수 있다.
- **유연한 UI**: 로그인 상태 변화에 따라 프로필 UI가 자동으로 업데이트되어 사용자 경험을 향상시킨다.
- **견고한 개발**: 테스트 파일을 통해 핵심 로직의 안정성을 미리 확보하여 잠재적인 버그를 줄인다.

### 사용 예시 (예상)

```jsx
// Header 컴포넌트 내에서 사용될 경우
import SearchBar from '../../atoms/SearchBar/SearchBar';
import ProfileCircle from '../../atoms/ProfileCircle/ProfileCircle';

function Header() {
  const { isLoggedIn, userProfileImage } = useAuth(); // 예시 훅

  return (
    <header>
      <SearchBar onSearch={handleSearch} />
      {isLoggedIn ? (
        <ProfileCircle imageUrl={userProfileImage} onClick={handleProfileClick} />
      ) : (
        <ProfileCircle onClick={handleLoginPrompt} /> // 로그인 유도
      )}
    </header>
  );
}
```
